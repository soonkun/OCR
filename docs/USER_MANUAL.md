# 고문서 OCR 복원 — 사용자 매뉴얼

저화질 고문서(한국어+한자) 스캔본을 텍스트로 복원하는 프로그램의 사용 설명서입니다.
처음 쓰는 분도 이 문서만 따라 하면 됩니다.

---

## 1. 설치 (최초 1회)

1. 프로젝트 폴더에서 **`setup.bat`** 을 한 번 실행합니다(가상환경과 필요한 프로그램을
   설치). 몇 분 걸릴 수 있습니다.
   - `고문서OCR-실행.bat`을 먼저 더블클릭해도, 설치가 안 돼 있으면 자동으로 설치를 시도합니다.
2. (선택) **한자 한글 병기**까지 쓰려면 OpenAI API 키가 필요합니다 → [6번](#6-한자-한글-병기-켜기) 참고.

> 인터넷이 막힌 PC라면 모델을 미리 받아 복사하는 방법은 README의 "오프라인 사용"을 보세요.

---

## 2. 실행

**`고문서OCR-실행.bat` 더블클릭** — 이게 전부입니다.

- 검은 창(서버)이 잠깐 뜨고, 잠시 후 **브라우저가 자동으로 열립니다.**
- 주소는 **http://localhost:8000/app/** 입니다(자동으로 안 열리면 직접 입력).
- **프로그램을 끄려면** 그 검은 "Gomunseo OCR Server" 창을 닫으면 됩니다.

> 더블클릭하면 이전에 떠 있던 서버가 있으면 자동으로 정리하고 새로 켭니다.

---

## 3. 화면 구성

| 영역 | 설명 |
|---|---|
| **왼쪽 사이드바** | 파일 올리기, 인식 언어 선택, 글씨 크기, 문서 목록, 엔진 상태 |
| **가운데~오른쪽** | 선택한 문서의 진행바, **전처리 미리보기**(이미지), **인식 결과**(편집창) |
| **우상단 `?`** | 사용법 안내 투어를 다시 봅니다 |

---

## 4. 문서 처리 흐름

### ① 파일 올리기
사이드바의 점선 상자에 이미지(JPG·PNG 등)나 **PDF**를 끌어다 놓거나, 상자를 눌러 선택합니다.
여러 개를 한 번에 올릴 수 있습니다.

### ② 전처리 조정 (팝업)
파일을 올리면 **전처리 조정** 창이 뜹니다.

- **처리본/원본** 탭으로 보정 전후를 비교합니다.
- **밝기·대비·감마·선명도·디노이즈** 슬라이더로 흐린 글자를 살립니다.
  (감마를 올리면 흐린 획이 살아나고, 디노이즈는 얼룩을 줄입니다.)
- **흑백·업스케일·기울기 보정·이진화** 토글. *이진화는 흐린 활자엔 보통 해로워서 기본 꺼짐.*
- **PDF는 `◀ N / 전체 ▶`** 로 페이지를 넘겨가며 여러 페이지를 미리볼 수 있습니다.
- **🔍 OCR 적합도 측정** — 가운데 일부만 빠르게 OCR해 신뢰도를 알려줍니다(설정이 좋은지 확인).
- 다 맞췄으면 **"이 설정으로 처리 시작"**. (여러 파일은 첫 장으로 맞춘 설정이 전체에 적용)

### ③ 처리 진행 확인
- 상단 **진행바**와 단계(`문자 인식 중 (3/54페이지)`)가 표시됩니다.
- **전처리 미리보기**에는 **지금 처리 중인 페이지**가 실시간으로 보여, 어디까지 왔는지 알 수 있습니다.
- 한국어+한자 모드 + API 키가 있으면 마지막에 **한글 병기 보정**이 진행됩니다.

### ④ 결과 보기 · 편집
완료되면:

- **전처리 미리보기** 위의 `◀ N / 전체 ▶` 로 페이지를 넘기면, **왼쪽 이미지와 오른쪽
  텍스트가 같이 그 페이지로** 이동합니다 → **이미지를 보며 그 페이지 텍스트를 편집**.
- **미리보기↔결과 사이의 막대를 좌우로 드래그**해 칸 폭을 조절합니다. 미리보기를 좁히면
  페이지가 한눈에 들어오고(스크롤 없이), 결과 칸을 넓혀 편집이 편해집니다.
  (막대를 **더블클릭**하면 50:50으로 초기화)
- 오른쪽 편집창에서 텍스트를 고치고 **"저장"** 을 누르면 반영됩니다.
- 우상단 **⬇ TXT** 로 결과를 텍스트 파일로 내려받습니다.

### ⑤ 삭제 · 다시 처리
- **삭제**: 문서를 지웁니다. **처리 중에 삭제하면 OCR도 곧바로 중단**됩니다.
- **다시 처리**: 설정/코드를 바꾼 뒤 같은 문서를 다시 돌립니다.

## 5. 편의 기능

- **글씨 크기**: 사이드바의 `A− / 슬라이더 / A+` 로 전체 UI를 90~130%로 키우거나 줄입니다(기억됨).
- **사용법 안내 투어**: 처음 열면 자동으로 안내가 뜨고, 우상단 **`?`** 로 언제든 다시 볼 수 있습니다.

---

## 6. 한자 한글 병기 켜기

한자를 `한자(漢字)`가 아니라 **`한글(漢字)`** 처럼 한글 독음과 함께 보여주는 보정은
**OpenAI API 키**가 있어야 동작합니다(없으면 한자 자리가 비거나 한자만 출력).

1. 프로젝트 폴더의 **`.env.example`** 파일을 복사해 이름을 **`.env`** 로 바꿉니다.
2. `.env` 를 열어 다음 한 줄을 본인 키로 채웁니다:
   ```ini
   OPENAI_API_KEY=sk-...
   ```
3. **서버를 다시 시작**합니다(런처 다시 더블클릭). 좌하단에 **"한글 병기 LLM 보정 켜짐"**
   이 보이면 성공입니다.

> `.env` 는 git에 올라가지 않으니 키가 외부에 노출되지 않습니다. 키는 OpenAI 사용량에
> 따라 비용이 발생합니다.

## 7. 자주 묻는 질문

**Q. 서버 창에 `GET /api/documents ...` 가 계속 떠요. 정상인가요?**
네. 화면이 진행률을 1초마다 확인하느라 보내는 요청입니다. 런처로 실행하면 이 로그는
숨겨져 깔끔하게 보입니다.

**Q. 화면을 새로 고쳐도 바뀐 기능이 안 보여요.**
브라우저 캐시 때문일 수 있습니다. **`Ctrl + Shift + R`**(강력 새로고침) 한 번 해보세요.

**Q. 처리가 너무 오래 걸려요.**
CPU로 도는 데다 한국어+한자(듀얼패스)는 페이지당 시간이 더 걸립니다(수십 초). 페이지가
많은 문서는 수십 분이 걸릴 수 있습니다. 진행바와 현재 페이지로 확인하세요.

**Q. 옛 문서라 글자가 틀리게 인식돼요.**
전처리 조정에서 감마·대비를 올리고 'OCR 적합도 측정'으로 더 나은 설정을 찾아보세요.
그래도 남는 오류는 결과 편집창에서 직접 고치면 됩니다.

**Q. 예전에 처리한 문서는 결과 화면에서 페이지 넘김 이미지가 안 보여요.**
페이지별 이미지 저장 기능이 추가되기 전에 처리된 문서입니다. **"다시 처리"** 를 한 번
돌리면 페이지별 이미지까지 모두 생깁니다.
